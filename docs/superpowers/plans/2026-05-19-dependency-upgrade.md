# Dependency Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade TauriScapes dependencies to current stable versions across pnpm, Vue/Vite, and Tauri/Rust while preserving current behavior and validating the updater release path.

**Architecture:** Execute the upgrade in staged layers: environment baseline, Tauri stable migration, Vue/runtime dependency migration, build/lint/style tooling migration, then release-path verification. Each layer refreshes the relevant lockfile and runs the narrowest useful checks before moving on.

**Tech Stack:** pnpm, Vue 3, Vue Router, Pinia, VueUse, Vite, TypeScript, ESLint with `@antfu/eslint-config`, Tailwind, DaisyUI, Tauri 2, Rust crates, Tauri updater.

---

## File Structure

- Modify `package.json`: update npm dependencies, scripts only if required by upgraded tools, and optional package manager or engine metadata if needed.
- Modify `pnpm-lock.yaml`: regenerated by `pnpm install`.
- Modify `src-tauri/Cargo.toml`: update Rust dependencies from RC/git to stable registry versions.
- Modify `src-tauri/Cargo.lock`: regenerated by Cargo/Tauri when Rust tooling is available.
- Modify `src-tauri/tauri.conf.json`: only if stable Tauri schema or updater artifact settings require compatibility changes.
- Modify `src-tauri/capabilities/default.json`: only if stable plugin permissions changed.
- Modify `src-tauri/src/main.rs`: only if stable Tauri or plugin APIs require code updates.
- Modify `src/stores/updater.ts`: only if `@tauri-apps/plugin-updater` stable API requires code updates.
- Modify `src/utils/request.ts`, `src/views/settings/index.vue`, `src/app.vue`, and other `src/**` files only when type checking identifies dependency API changes.
- Modify `vite.config.ts`, `tailwind.config.js`, `postcss.config.cjs`, `tsconfig.json`, or `tsconfig.node.json` only when upgraded tooling requires config migration.
